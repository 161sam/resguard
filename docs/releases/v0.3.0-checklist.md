## Go/No-Go Criteria

All criteria below must be `PASS` before cutting `v0.3.0`.

### 1) Suggest tested on GNOME/KDE

- [ ] GNOME Wayland: `resguard suggest --dry-run` produces stable, reviewable output
- [ ] GNOME X11: `resguard suggest --dry-run` produces stable, reviewable output
- [ ] KDE Wayland: `resguard suggest --dry-run` produces stable, reviewable output
- [ ] KDE X11: `resguard suggest --dry-run` produces stable, reviewable output
- [ ] No crashes/timeouts when user manager has many scopes

Evidence to capture:

- command output snippet
- environment (`XDG_CURRENT_DESKTOP`, `XDG_SESSION_TYPE`, kernel)
- result file from `tests/e2e/results/`

### 2) Confidence threshold is sane

- [ ] `--confidence-threshold` rejects invalid values (`<0`, `>100`)
- [ ] threshold `70` suppresses weak matches while keeping obvious matches
- [ ] threshold `90+` only applies high-confidence suggestions
- [ ] threshold `0` includes all suggestions without parser/format regressions

Evidence to capture:

- before/after suggestion counts at `0`, `70`, `90`
- at least one GNOME run + one KDE run

### 3) Wrap integration validated

- [ ] `resguard suggest --apply` wraps only entries with unique desktop ID mapping
- [ ] non-unique mappings emit manual hint instead of forcing wraps
- [ ] wrapped desktop entries launch through `resguard run --class ...`
- [ ] `resguard desktop doctor` reports expected state and fix hints

Evidence to capture:

- mapping file diff (`~/.config/resguard/desktop-mapping.yml`)
- wrapper desktop file example (`~/.local/share/applications/*.resguard-*.desktop`)
- launch verification on at least one app per desktop environment

### 4) No accidental overwrite outside user app dir

- [ ] wrap path remains under `~/.local/share/applications/` in normal mode
- [ ] invalid desktop IDs (`../`, `/`, malformed) are rejected
- [ ] unwrap does not delete arbitrary files when mapping path is tampered
- [ ] override mode remains explicit and guarded (`--override` + `--force`)

Evidence to capture:

- negative test outputs for invalid IDs/paths
- confirmation that only expected wrapper/backup files are touched

## Known Risks

- Desktop environments may bypass wrapper `Exec` when `DBusActivatable=true` is preferred.
- Heuristic classification can overfit high-memory bursts and suggest wrong classes.
- User-manager visibility differs across sessions (TTY vs desktop login), reducing suggestion signal quality.
- Wrapper mapping drift can occur if users manually edit/remove files outside the tool flow.

## Manual Validation Points

1. Run E2E kit:

```bash
tests/e2e/run_e2e.sh --profile e2e-field --class rescue --yes
```

2. Run suggest threshold sweep:

```bash
resguard suggest --dry-run --confidence-threshold 0
resguard suggest --dry-run --confidence-threshold 70
resguard suggest --dry-run --confidence-threshold 90
```

3. Validate apply path:

```bash
resguard suggest --apply --confidence-threshold 70
resguard desktop doctor
```

4. Validate path safety quickly:

```bash
resguard desktop wrap ../bad.desktop --class browsers
resguard desktop wrap firefox.desktop --class browsers --dry-run
```

## Exit Condition

`v0.3.0` is release-ready only when every Go/No-Go checkbox is complete with captured evidence and no open high-severity wrapper/suggest defects remain.
