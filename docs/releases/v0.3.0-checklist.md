### 2) Confidence threshold is sane

Evidence to capture:

### 3) Wrap integration validated

- [x] wrapped desktop entries launch through `resguard run --class ...` in the documented Firefox snap flow

Evidence to capture:

- [x] wrap path remains under `~/.local/share/applications/` in normal mode
- [x] invalid desktop IDs (`../`, `/`, malformed) are rejected

Evidence to capture:

- confirmation that only expected wrapper/backup files are touched

## Known Risks

- Desktop environments may bypass wrapper `Exec` when `DBusActivatable=true` is preferred.
- Heuristic classification can overfit high-memory bursts and suggest wrong classes.
- User-manager visibility differs across sessions (TTY vs desktop login), reducing suggestion signal quality.
- Wrapper mapping drift can occur if users manually edit/remove files outside the tool flow.
- Full cross-desktop validation matrix (GNOME X11, KDE Wayland/X11) is not yet captured in `tests/e2e/results/`.

## Manual Validation Points

1. Run E2E kit:

```bash
tests/e2e/run_e2e.sh --profile e2e-field --class rescue --yes
```

2. Run suggest threshold sweep:

```bash
resguard suggest --dry-run --confidence-threshold 0
resguard suggest --dry-run --confidence-threshold 70
resguard suggest --dry-run --confidence-threshold 90
```

3. Validate apply path:

```bash
resguard suggest --apply --confidence-threshold 70
resguard desktop doctor
```

4. Validate path safety quickly:

```bash
resguard desktop wrap ../bad.desktop --class browsers
resguard desktop wrap firefox.desktop --class browsers --dry-run
```
