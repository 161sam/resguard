# Resguard v0.2.1 Release Notes

## Highlights

- Desktop wrapper workflows added:
  - `resguard desktop list`
  - `resguard desktop wrap`
  - `resguard desktop unwrap`
  - `resguard desktop doctor`
- Wrapper safety hardening:
  - strict desktop-id validation
  - controlled wrapper path handling
  - safer unwrap/restore behavior
- Diagnostics improvements:
  - `resguard doctor` includes desktop mapping checks and actionable fixes
- Packaging/release consistency updates:
  - version alignment checks between Cargo crates and Debian control metadata

## Install Instructions

### Debian package

```bash
sudo dpkg -i resguard_0.2.1_amd64.deb
```

If dependencies are missing:

```bash
sudo apt-get -f install
```

### Build from source

```bash
cargo build --release -p resguard
```

Binary path:

- `target/release/resguard`

## Upgrade Notes

- Existing profiles remain compatible with `0.2.1`.
- If you use desktop wrappers, run:

```bash
resguard desktop doctor
```

- If user slices changed after apply, reload user daemon in the active session:

```bash
systemctl --user daemon-reload
```

## Known Limitations

- `diff` and several `profile` subcommands remain stubs.
- TUI is still feature-gated and not enabled by default.
- `resguardd` remains optional and is not always-on by default.
- Wrapper usage can vary across desktop environments when `DBusActivatable=true` entries are involved.
